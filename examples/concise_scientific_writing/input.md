# Example Technical Note

It should be noted that this repository is intended to provide a small demonstration of an auditable document transformation workflow. The workflow exists in order to show how observations can become recommendations and then transformations.

The current implementation has the ability to perform an analysis of a document due to the fact that it uses a small set of explicit phrase transformations. It preserves references such as (Hotaling, 2020) and numbers such as 42.

This sentence is intentionally long because the harness should be able to detect long sentences that may increase reader effort even when it does not automatically rewrite them during the first conservative version of the system.
